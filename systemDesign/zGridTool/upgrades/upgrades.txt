Not a dumb question at all. Right now our `zGridTool` package only draws the z-grid (plus pole/zero overlays) — no root-locus, Nyquist, or Nichols yet.

### How hard is it to add them?

**Reasonable** effort, not a rewrite. We already have:

* a clean CLI, request/response dataclasses, and an app façade
* math helpers for z-plane geometry
* consistent in/out conventions and CSV/JSON overlay parsing

What we need to add is:

* a minimal **LTI model layer** (to represent G(z) from (num, den))
* routines to compute **closed-loop poles vs K** (root locus)
* routines to compute **frequency response on the unit circle** (Nyquist/Nichols)
* plotters for those results (MPL and optional Plotly)

That fits our OOP/TDD style very nicely.

## Minimal algorithms (discrete-time)

### Root locus (z-plane)

For unity feedback, closed-loop char poly:

```
D_cl(z; K) = den(z) + K * num(z)
```

For each K on a grid (e.g., lin/log), compute roots of `D_cl(z; K)`; those are the loci. Then:

* overlay asymptotes (optional)
* overlay unit circle (for discrete-time stability)
* annotate breakaway points (optional later)

### Nyquist (discrete)

Use the **unit circle**: `z = e^{j ω}`, `ω ∈ [0, π]` (or `[0, 2π]` for convenience).
Compute `H(e^{jω}) = Num(e^{jω})/Den(e^{jω})` and plot Re vs Im.

### Nichols

Transform the same `H(e^{jω})`:

* `mag_db = 20 log10 |H|`, `phase_deg = unwrap(angle(H)) * 180/π`
* Plot `phase_deg` vs `mag_db`. (Optionally overlay Nichols grid later.)

# Root locus (discrete-time, z-plane)
python cli.py rootlocus \
  --num "0.5 0.1" --den "1 -0.8" \
