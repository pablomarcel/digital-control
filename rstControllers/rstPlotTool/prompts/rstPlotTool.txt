where script_1, script_2, script_3 represent the meat and potatoes of the kalmanFilter refactored package.
Basically we are breaking up kalman_filter.py into several scripts, by separation of concerns following a strict object-oriented approach with test driven development in mind.
The code must be absolutely testable.
The code must be scalable. For example, if we want to implement more options or flags.

for the kalman filter tool package, any input files would be stored in kalmanFilters/kalmanFilter/in
for the kalman filter tool package, any output files would be stored in kalmanFilters/kalmanFilter/out

use decorators as applicable.

test will be stored in kalmanFilters/kalmanFilter/tests

any tools will be stored in kalmanFilters/kalmanFilter/tools

So we are going to build a beautiful architecture, with crisp classes and methods and all the goodies that come with oop approach.
The proposed architecture for the kalmanFilter package would be something like this:
